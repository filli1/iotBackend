# DATA-01: Session History Table — Design Spec

**Date:** 2026-04-04
**Backlog item:** DATA-01
**Status:** Draft

---

## Goal

A paginated, sortable, filterable table of all past presence sessions. Allows staff to review who engaged with a product display, for how long, and whether they picked up the product.

---

## Approach

A `/history` frontend page that fetches sessions from a REST endpoint with pagination, sorting, and filtering applied server-side. No client-side data manipulation — the backend does all filtering and ordering via Prisma queries.

---

## Backend — REST Endpoint

```
GET /api/sessions
```

### Query Parameters

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | number | 1 | 1-based page number |
| `pageSize` | number | 25 | Rows per page (max 100) |
| `sortBy` | string | `startedAt` | Column to sort by |
| `sortDir` | `asc\|desc` | `desc` | Sort direction |
| `unitId` | string | — | Filter by unit |
| `dateFrom` | ISO string | — | Sessions starting on or after |
| `dateTo` | ISO string | — | Sessions starting before |
| `minDwellSeconds` | number | — | Filter sessions shorter than this out |
| `productPickedUp` | boolean | — | Filter by pickup |
| `status` | `completed\|active` | `completed` | Only return completed sessions by default |

### Response

```typescript
{
  data: Array<{
    id: string
    unitId: string
    unitName: string
    startedAt: string
    endedAt: string
    dwellSeconds: number
    productPickedUp: boolean
  }>
  total: number
  page: number
  pageSize: number
  pageCount: number
}
```

### Implementation

Prisma `findMany` with `where`, `orderBy`, `skip`, and `take`. `unitName` is joined from `SensorUnit`. Total count uses a separate `count` call with the same `where`.

## Frontend — History Page

```
/history
```

### Layout

```
┌─────────────────────────────────────────────────┐
│  Session History                                │
│                                                 │
│  [Unit ▾] [Date from] [Date to] [Min dwell]     │
│  [Pickup: All ▾]          [Export CSV ↓]        │
├─────────┬──────────┬────────┬─────────┬─────────┤
│ Unit    │ Started  │ Dwell  │ Pickup  │ Duration│
├─────────┼──────────┼────────┼─────────┼─────────┤
│ unit-01 │ 14:32:01 │ 47s    │ Yes     │         │
│ unit-01 │ 14:15:44 │ 12s    │ No      │         │
│ ...     │ ...      │ ...    │ ...     │         │
├─────────┴──────────┴────────┴─────────┴─────────┤
│  [← Prev]  Page 1 of 12  [Next →]              │
└─────────────────────────────────────────────────┘
```

### Columns

| Column | Sortable | Notes |
|--------|----------|-------|
| Unit | Yes | Shows unit name |
| Started | Yes | Local datetime |
| Dwell | Yes | Formatted as `Xs` or `Xm Ys` |
| Pickup | Yes | "Yes" / "No" |

### Filter Behaviour

Filters trigger a new fetch with `page = 1` reset. Filters are stored in URL query params so the page is bookmarkable and shareable.

---

## Dwell Duration Formatting

```typescript
function formatDwell(seconds: number): string {
  if (seconds < 60) return `${seconds}s`
  return `${Math.floor(seconds / 60)}m ${seconds % 60}s`
}
```

## Dependencies

No new dependencies. React Router for URL params, native `fetch` for REST calls.

---

## Acceptance Criteria

- [ ] `GET /api/sessions` returns paginated results with correct total/pageCount.
- [ ] Sorting by any column changes the result order.
- [ ] Date range filter correctly restricts results.
- [ ] `productPickedUp=true` filter returns only sessions with product pickup.
- [ ] `minDwellSeconds` filter removes short sessions.
- [ ] Filters are reflected in the URL and survive page refresh.
- [ ] Table is readable on a 1280×800 tablet.
- [ ] Sessions with `status: "active"` are excluded from results by default.

---

## Out of Scope

- CSV export (DATA-03)
- Clicking a row to view session events in detail
- Bulk deletion of sessions
