# DASH-01: Live Sensor Dashboard — Design Spec

**Date:** 2026-04-04
**Backlog item:** DASH-01
**Status:** Draft

---

## Goal

A real-time frontend page showing the live state of all registered sensor units: connection status, which ToF sensors are detecting, current distances, PIR state, and IMU state. Updates via WebSocket at ~2 Hz.

---

## Approach

A single `/dashboard` page composed of per-unit `SensorUnitCard` components. A shared `useWebSocket` hook manages the WebSocket connection and feeds a global state store (Zustand or React context). Each card subscribes to its unit's slice of state.

---

## Backend — WebSocket Endpoint

Added to `backend/src/ws/broadcaster.ts` and registered in `backend/src/index.ts`.

```
GET ws://localhost:7000/ws
```

The backend broadcasts two message categories to all connected clients:

**Sensor reading** (forwarded from ingest route, ~2 Hz per unit):
```typescript
{
  type: "sensor_reading"
  unitId: string
  ts: string
  tof: Array<{ id: number; distance_mm: number; status: "valid" | "out_of_range" | "error" }>
  pir: { triggered: boolean; last_trigger_ms: number }
  imu: { accel: XYZ; gyro: XYZ; mag: XYZ }
  presenceState: "idle" | "pending" | "active" | "departing"
}
```

**Session event** (from SessionManager, on state changes):
```typescript
{
  type: "session_event"
  unitId: string
  event: "session_started" | "session_ended" | "product_picked_up" | "product_put_down"
  sessionId: string
  dwellSeconds?: number
  productPickedUp?: boolean
  ts: string
}
```

**Unit status** (from unit registry, when online/offline changes):
```typescript
{
  type: "unit_status"
  unitId: string
  status: "online" | "offline"
  lastSeen: string
}
```

**Alert fired** (from SessionManager):
```typescript
{
  type: "alert_fired"
  unitId: string
  sessionId: string
  reason: "dwell_threshold" | "pickup" | "dwell_and_pickup"
  ts: string
}
```

## Frontend State

`frontend/src/lib/wsStore.ts` — a Zustand store updated by the WebSocket hook:

```typescript
type UnitLiveState = {
  unitId: string
  status: "online" | "offline"
  lastSeen: string | null
  presenceState: "idle" | "pending" | "active" | "departing"
  tof: TofReading[]
  pir: PirState | null
  imu: ImuState | null
  lastEvent: SessionEvent | null
}

type WsStore = {
  units: Record<string, UnitLiveState>
  connected: boolean
}
```

## Frontend Components

```
frontend/src/pages/DashboardPage.tsx
  └─ frontend/src/components/SensorUnitCard.tsx
       ├─ frontend/src/components/TofGrid.tsx      (6-sensor arc/grid)
       ├─ frontend/src/components/PirBadge.tsx
       └─ frontend/src/components/ImuBadge.tsx
frontend/src/hooks/useWebSocket.ts               (manages WS connection + feeds store)
frontend/src/lib/wsStore.ts                      (Zustand store)
```

### SensorUnitCard

Displays per unit:
- Header: unit name, location, online/offline badge (green/red dot)
- Presence state pill: `Idle` / `Detecting…` / `Person Present` / `Leaving…`
- `TofGrid`: 6 coloured cells, one per ToF sensor. Colour: grey = out_of_range/error, blue = valid but far, green = detecting (within threshold)
- `PirBadge`: triggered / idle
- `ImuBadge`: idle / shock / pickup / rotation
- Last event (e.g., "Product picked up 12s ago")

### TofGrid

Renders 6 sensor cells in a 3×2 grid (or arc layout). Each cell shows:
- Sensor label (from `TofSensor.label`)
- Distance in mm (or "—" if out_of_range)
- Background colour indicating state

### useWebSocket

- Opens `ws://localhost:7000/ws` on mount
- Reconnects with exponential backoff (1s, 2s, 4s, max 30s) on disconnect
- Shows a floating "Reconnecting…" banner when not connected
- Dispatches parsed messages to the Zustand store

## Routing

Add `/dashboard` route to the React app using React Router:

```
/ → redirect to /dashboard
/dashboard → DashboardPage
```

---

## Tablet Layout

- Designed for landscape ~10" (1280×800)
- Grid of unit cards: 1–2 columns depending on unit count
- Font size for distances: 18px minimum (readable at arm's length)
- No horizontal scroll

---

## Acceptance Criteria

- [ ] `GET ws://localhost:7000/ws` establishes a WebSocket connection.
- [ ] A sensor reading POSTed to the ingest endpoint appears on the dashboard within 600ms.
- [ ] Each ToF sensor cell reflects correct colour based on distance vs threshold.
- [ ] Unit card shows "Offline" when no reading received for >60s.
- [ ] WebSocket disconnection shows a reconnecting banner; dashboard recovers automatically.
- [ ] Dashboard is usable on a 1280×800 landscape tablet.
