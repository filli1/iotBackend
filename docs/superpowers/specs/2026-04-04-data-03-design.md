# DATA-03: CSV Export — Design Spec

**Date:** 2026-04-04
**Backlog item:** DATA-03
**Status:** Draft

---

## Goal

Allow users to export session history as a CSV file from the session history table (DATA-01). The export respects the currently active filters.

---

## Approach

A backend endpoint streams all matching sessions (no pagination limit) as CSV text. The frontend triggers a file download by navigating to the URL with the current filter params. No CSV library needed — CSV generation is simple enough to hand-roll.

---

## Backend — Endpoint

```
GET /api/sessions/export.csv
```

Accepts the same filter query params as `GET /api/sessions` (minus `page`, `pageSize`, `sortBy`, `sortDir` — export always returns all matching rows, sorted by `startedAt desc`).

Response:
- `Content-Type: text/csv`
- `Content-Disposition: attachment; filename="sessions-<date>.csv"`
- Body: CSV rows streamed directly to the response

### CSV Format

```
id,unitId,unitName,startedAt,endedAt,dwellSeconds,productPickedUp
"sess_abc","unit-01","Product Stand A","2026-04-04T14:32:01.000Z","2026-04-04T14:32:48.000Z",47,true
...
```

All fields are quoted. Boolean `productPickedUp` exported as `true`/`false`. Timestamps in ISO 8601 UTC.

### Implementation

```typescript
// backend/src/routes/sessions.ts — add export handler

const rows = await prisma.presenceSession.findMany({
  where: buildWhereClause(query),
  orderBy: { startedAt: 'desc' },
  include: { unit: { select: { name: true } } },
})

const header = 'id,unitId,unitName,startedAt,endedAt,dwellSeconds,productPickedUp\n'
const body = rows.map(r =>
  `"${r.id}","${r.unitId}","${r.unit.name}","${r.startedAt.toISOString()}","${r.endedAt?.toISOString() ?? ''}",${r.dwellSeconds},${r.productPickedUp}`
).join('\n')

reply
  .header('Content-Type', 'text/csv')
  .header('Content-Disposition', `attachment; filename="sessions-${new Date().toISOString().slice(0, 10)}.csv"`)
  .send(header + body)
```

No streaming required at prototype scale (SQLite, expected <10k rows).

## Frontend — Export Button

The "Export CSV ↓" button in the session history table (DATA-01) constructs the export URL from the current filter state and opens it:

```typescript
function handleExport() {
  const params = new URLSearchParams(currentFilters)
  window.open(`/api/sessions/export.csv?${params}`, '_blank')
}
```

The browser handles the file download automatically via `Content-Disposition: attachment`.

The export button is already included in the DATA-01 layout spec. This item implements the backend handler and the button's click handler.

## Acceptance Criteria

- [ ] `GET /api/sessions/export.csv` returns a valid CSV with a header row and one row per matching session.
- [ ] The file downloads with a dated filename (e.g. `sessions-2026-04-04.csv`).
- [ ] Active filters (date range, unit, pickup) are applied to the export.
- [ ] Empty result (no matching sessions) returns a header row only — no error.
- [ ] CSV opens correctly in Excel and Google Sheets (fields quoted, UTF-8).
- [ ] Export button in the history page triggers the download.

---

## Out of Scope

- Chart or analytics export as PNG/PDF
- Scheduling automatic exports
- Exporting raw `SessionEvent` rows (only the session summary is exported)
