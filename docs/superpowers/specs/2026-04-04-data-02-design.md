# DATA-02: Aggregate Analytics Dashboard — Design Spec

**Date:** 2026-04-04
**Backlog item:** DATA-02
**Status:** Draft

---

## Goal

A dashboard page showing aggregated session metrics over time: visitor count, average dwell time, product pickup rate, and peak traffic hours. Gives staff and managers a quick overview of engagement trends.

---

## Approach

A `/analytics` frontend page that fetches pre-aggregated stats from backend REST endpoints. Aggregations are computed via `prisma.$queryRaw` SQL queries (SQLite-compatible). Charts are rendered with Recharts (lightweight, React-native). No separate time-series database needed for prototype scale.

## Backend — REST Endpoints

All aggregations return only `completed` sessions.

### Summary Stats

```
GET /api/analytics/summary?unitId=&from=&to=
```

Response:
```typescript
{
  totalSessions: number
  avgDwellSeconds: number
  pickupRate: number       // 0–1
  avgDwellWithPickup: number
}
```

SQL: `COUNT`, `AVG(dwellSeconds)`, `SUM(CASE WHEN productPickedUp THEN 1 ELSE 0 END)`.

---

### Daily Session Counts

```
GET /api/analytics/daily?unitId=&from=&to=
```

Response:
```typescript
{
  data: Array<{ date: string; sessions: number; pickups: number }>
}
```

SQL: `GROUP BY date(startedAt)`.

---

### Peak Hours Heatmap

```
GET /api/analytics/heatmap?unitId=&from=&to=
```

Response:
```typescript
{
  data: Array<{ dayOfWeek: number; hour: number; sessions: number }>
  // dayOfWeek: 0=Sun–6=Sat, hour: 0–23
}
```

SQL: `GROUP BY strftime('%w', startedAt), strftime('%H', startedAt)`.

---

### Average Dwell Over Time

```
GET /api/analytics/dwell-trend?unitId=&from=&to=
```

Response:
```typescript
{
  data: Array<{ date: string; avgDwellSeconds: number }>
}
```

SQL: `AVG(dwellSeconds) GROUP BY date(startedAt)`.

## Charts (Recharts)

| Chart | Type | Data source |
|-------|------|-------------|
| Visitors per day | `BarChart` | `/api/analytics/daily` |
| Peak hours | custom grid (CSS, not Recharts) | `/api/analytics/heatmap` |
| Avg dwell trend | `LineChart` | `/api/analytics/dwell-trend` |

The heatmap is a 7×24 grid rendered with CSS Grid. Each cell is coloured from white → deep blue based on session count relative to the max cell in the dataset.

## Acceptance Criteria

- [ ] Summary stats are correct for the selected period and unit filter.
- [ ] Daily bar chart shows one bar per day in the selected range.
- [ ] Heatmap shows relative traffic by hour and day of week.
- [ ] All charts update when the unit or period filter changes.
- [ ] Queries return within 1 second for up to 90 days of data.
- [ ] Filters are reflected in the URL and survive page refresh.
- [ ] Charts are readable on a 1280×800 tablet.
