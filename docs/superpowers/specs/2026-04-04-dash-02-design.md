# DASH-02: Live Calibration Mode — Design Spec

**Date:** 2026-04-04
**Backlog item:** DASH-02
**Status:** Draft

---

## Goal

A frontend view that shows raw distance readings from all 6 ToF sensors in large, easy-to-read numbers. Used during physical installation to verify sensor angles and range thresholds. Must work on a phone held next to the display.

---

## Approach

A dedicated `/calibrate/:unitId` route that subscribes to the same WebSocket as the main dashboard but renders a stripped-down, phone-optimised layout. No new backend work required — the ingest endpoint already forwards readings over WebSocket.

---

## Route

```
/calibrate/:unitId
```

The unit selector is in the URL so the URL can be bookmarked or opened directly on a phone.

---

## Layout

Portrait and landscape phone (375px–430px wide). Large font sizes readable at arm's length.

```
┌─────────────────────────────┐
│  Store Attention — Calibrate│
│  unit-01  [online ●]        │
├─────────────────────────────┤
│  LEFT-WIDE      823 mm  ▓▓▓ │
│  LEFT           790 mm  ▓▓▓ │
│  CENTER-LEFT    ---      ░░░ │
│  CENTER-RIGHT   812 mm  ▓▓▓ │
│  RIGHT          ---      ░░░ │
│  RIGHT-WIDE     ---      ░░░ │
├─────────────────────────────┤
│  PIR: triggered             │
│  IMU: idle                  │
├─────────────────────────────┤
│  [← Back to Dashboard]      │
└─────────────────────────────┘
```

Each row shows:
- Sensor label (from `TofSensor.label`, uppercased)
- Distance in mm, or `---` if `status !== "valid"`
- A simple bar (CSS progress) coloured: green = within configured threshold, orange = near threshold, grey = out of range

The bar gives an instant visual cue without the installer needing to know the configured threshold value.

---

## Bar Colour Logic

```
distance <= maxDist × 0.75  → green (well within range)
distance <= maxDist          → orange (near threshold edge)
status !== "valid"           → grey (out of range / error)
```

`maxDist` per sensor comes from the unit's `TofSensor` configuration, fetched once via REST on page load (`GET /api/units/:unitId/sensors`).

---

## Data Flow

1. On mount: fetch `TofSensor` config for the unit via `GET /api/units/:unitId/sensors` (new REST endpoint)
2. Subscribe to the WebSocket store (same `useWebSocket` hook from DASH-01)
3. Filter messages for this `unitId`
4. Re-render on each `sensor_reading` message (~2 Hz)

---

## Backend — New REST Endpoint

```
GET /api/units/:unitId/sensors
```

Returns the `TofSensor` rows for the unit so calibration mode can compute bar colours:

```typescript
// Response
{
  sensors: Array<{
    index: number
    label: string
    minDist: number
    maxDist: number
  }>
}
```

## Dependencies

No new dependencies. Uses the WebSocket hook and Zustand store from DASH-01.

---

## Acceptance Criteria

- [ ] `/calibrate/unit-01` loads and shows 6 sensor rows.
- [ ] Each row updates within 600ms of a new sensor reading arriving.
- [ ] Rows show `---` for sensors with `status !== "valid"`.
- [ ] Bar colour reflects distance vs configured threshold.
- [ ] Page is usable on a 375px-wide phone in portrait mode with no horizontal scroll.
- [ ] "Back to Dashboard" link navigates to `/dashboard`.
- [ ] Unknown `unitId` shows a "Unit not found" error state.

---

## Out of Scope

- Editing thresholds inline (that is SETUP-02)
- Showing IMU raw values (accel/gyro numbers) — just the classified state label
- Recording calibration sessions
